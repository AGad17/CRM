03 Roles & Permissions

Every user has one system role. Admins can also create custom roles as reusable permission templates.

👑 CCO_ADMIN

Full access to all modules and settings. Manages users, custom roles, and pricing. Cannot be restricted by custom roles.

💼 REVENUE_MANAGER

Write access to pipeline, accounts, and invoicing. Read access to analytics. Can update settings but not delete.

🤝 CUSTOMER_SUCCESS

Full access to Operations (onboarding). Read-only on Pipeline, Invoicing, and Analytics. No settings access.

👁 READ_ONLY

View-only across all modules. Default for new users until a role is assigned.

Permission Layers

Permissions resolve in 3 layers — higher layers override lower ones:

// Layer 1: System role defaults ROLE_DEFAULTS[user.role] → base permissions // Layer 2: Custom role (replaces Layer 1 if assigned) CustomRole.permissions → overrides base // Layer 3: Per-user overrides (applied on top) User.permissions JSON → fine-tuning exceptions

04 Dashboard

📊
Who uses it: Everyone  ·  Permission: dashboard.view
Real-time executive snapshot. Use the Lead Source and Country filters at the top to narrow all KPIs.

Account KPIs

KPIDefinition
Total AccountsAll accounts ever created
Active AccountsAccounts with at least one valid, non-cancelled contract
Expired AccountsAll contracts past end date, no cancellation (natural lapse)
Churned AccountsAccounts with at least one explicitly cancelled contract
Overall Churn RateChurned ÷ Total Accounts × 100

Expired vs. Churned: Expired = contract ended naturally (renewal opportunity). Churned = customer explicitly cancelled (harder to recover).

07 Operations (Customer Journey)

🏗️
Who uses it: Customer Success (primary), Revenue Manager, CCO_ADMIN  ·  Permission: onboarding.view
Tracks every customer through a structured 7-phase lifecycle from deal close to ongoing management (or churn).

The 7 Phases

#PhaseOwnerDurationKey Goal
1Deal ClosureSales / AM~3 daysContract signed, SOP shared, internal handover, kickoff email sent
2OnboardingOnboarding Team~23 daysSystem configured, menus/data loaded, QA passed, client presentation done
3TrainingOnboarding Team~14 daysAll modules trained (Inventory, Recipes, Accounting, Production, Forecasting, Reports), go-live ready
4IncubationOnboarding Team~14 daysDay-1 go-live support, daily logs, mid-review, CSAT captured, closure report signed
5Account ManagementCustomer SuccessOngoingMonthly check-ins, quarterly reviews, upsell identification. Recurring tasks auto-generate.
6ExpiredCustomer SuccessUntil renewedContract lapsed naturally — pursue renewal. Shows days until deadline.
7Churned—TerminalExplicit cancellation. Removed from active reporting. No further automated actions.

Auto-Sync Logic

The Operations page automatically runs syncExpiredTrackers() on every load:

  • If all contracts are past end date and none were cancelled → moves tracker to Expired
  • If any contract was cancelled → tracker stays in or moves to Churned
✅
Contract data is always the source of truth. Click Sync All Accounts manually to create trackers for any accounts missing one.

Kanban View

7-column kanban — one per phase. Each card shows:

  • Account name + country
  • "X days in phase" badge: <7d 7–14d >14d
  • Overdue task count, package tags, task progress bar

Tracker Detail Page

Click any account card to open the full tracker:

  1. 1Account & Deal Info — linked account, contract summary
  2. 2Current Phase & Progress — visual phase bar, % tasks completed
  3. 3Task Checklist — grouped by phase; toggle completion; overdue = red; recurring tasks auto-generate on completion
  4. 4Staff Assignments — Onboarding Specialist, Training Specialist, Account Manager
  5. 5CSAT History — auto-created per phase transition, 1–5 scale
  6. 6NPS History — quarterly for Incubation + Account Mgmt phases, 0–10 scale
  7. 7Notes Timeline — timestamped team notes with author

11 How Numbers Are Calculated

MRR & ARR

Contract MRR = Annual Contract Value ÷ 12 Total MRR = Sum of all active Contract MRRs ARR = Total MRR × 12 // "Active" = cancellationDate IS NULL AND endDate ≥ today

ACV (Annual Contract Value)

ACV = Contract Value ÷ Contract Months × 12

NRR & GRR

NRR = (End MRR + Expansion MRR − Churned MRR) ÷ Start MRR × 100 GRR = (End MRR − Churned MRR) ÷ Start MRR × 100 // NRR > 100% = net expansion (good). GRR excludes expansion.

Churn Rate

Churn Rate % = Churned Accounts ÷ Total Accounts × 100 // "Churned" = accounts with cancellationDate IS NOT NULL

Account Health Score (0–100)

Factors: + Task completion % in current phase + Average CSAT score (1–5, normalized) + Average NPS score (0–10, normalized) − Deductions per overdue task

Deal & Invoice Math

Base MRR = Branch Pricing × Branch Count (per type) Add-on MRR = Add-on Pricing × Unit Count Total MRR = (Base + Add-on) × (1 − Discount%) VAT Amount = Total MRR × Country VAT Rate Total incl VAT = Total MRR + VAT Amount Contract Value = Total MRR (excl VAT) × Contract Months Foodics Eligible Date = Invoice Date + 38 days

Days in Phase Coloring

RangeColorAction
< 7 daysGreenFresh start — no action needed
7–14 daysAmberMonitor closely
> 14 daysRedInvestigate — account may be stuck

12 Common Workflows

Workflow 1 — New Customer (End-to-End)

✨ New Deal → Fully Onboarded Customer
  1. 1Create Lead in Pipeline → Type: New, Channel: [source]
  2. 2Qualify the Lead — update contact info and estimated value
  3. 3Move to Closed Won → go to Invoicing → Deal Calculator → configure and confirm
  4. 4Account + Contract + Invoice(s) created automatically
  5. 5Onboarding Tracker auto-created in Deal Closure phase — assign Onboarding Specialist
  6. 6Advance through phases: Deal Closure → Onboarding → Training → Incubation → Account Management
  7. 7CSAT is auto-captured at each phase advance. NPS recorded quarterly.

Workflow 2 — Contract Renewal

🔄 Renewing an Existing Customer
  1. 1Create Lead → Type: Renewal, link to existing Account
  2. 2Qualify and move to Closed Won
  3. 3In Deal Calculator, set Deal Type = Renewal → new contract and invoice(s) created
  4. 4Account remains in Account Management phase in Operations

Workflow 3 — Expansion / Upsell

📈 Adding Revenue to an Existing Account
  1. 1Create Lead → Type: Expansion, link to existing Account
  2. 2Qualify and close
  3. 3In Deal Calculator, set Deal Type = Expansion or Upsell → MRR increase recorded as Expansion MRR in the waterfall

Workflow 4 — Invoicing a Foodics Deal

💳 Tracking a Foodics Invoice to Collection
  1. 1Deal confirmed → Invoice created, Status = Pending
  2. 2After 38 days → Go to Invoicing → Foodics AR, change status to Eligible
  3. 3When Foodics pays → enter Foodics Invoice #, set status to Collected, enter collection date
  4. 4Invoice moves to Foodics History; collection cycle days recorded automatically

Workflow 5 — Handling an Expired Account

⏰ Pursuing a Naturally Lapsed Account
  1. 1Operations auto-sync moves the account to Expired when contract end date passes with no cancellation
  2. 2View in Operations → Expired column
  3. 3To pursue renewal: create new Lead (Type: Renewal) linked to the account
  4. 4If renewed: new contract → MRR resumes → account returns to Account Management

Workflow 6 — Setting Up a New Team Member

👤 Onboarding a New User
  1. 1Settings → Roles → + Create Role (if a custom role is needed)
  2. 2Settings → Team → + Invite User → enter email, name, initial role
  3. 3Assign custom role via the Custom Role dropdown in the Team tab
  4. 4Fine-tune with per-user overrides using the ✏️ Permissions button if needed
